Group Definition

Each entry: {"field": "...", "operator": "...", "value": ...}. All entries must match for a system to join the group. See sidebar for fields, operators, and examples.
Cancel

Reference

Fields:

  • hostname, ip, os, os_family, arch, platform
  • ver (agent version — semver compare)
  • status (active / pending / …)
  • mem_total_mb, disk_total_gb, uptime_secs
  • containers_exists (bool)
  • has_runtime (docker / podman)
  • any_container_image

Operators:

  • string: equals, not_equals, contains, not_contains, starts_with, ends_with, regex
  • numeric / semver: eq, ne, lt, le, gt, ge
  • set: in, not_in
  • IP: in_cidr

Example — Linux web prod:

[
  {"field":"os_family","operator":"equals","value":"linux"},
  {"field":"hostname","operator":"regex","value":"^web-prod-"},
  {"field":"ip","operator":"in_cidr","value":"10.10.0.0/16"}
]

Example — Hosts with Docker:

[
  {"field":"has_runtime","operator":"equals","value":"docker"}
]
